Summary

Our ticket booking management system project underwent a thorough development process, beginning with drafting the Entity Relationship Diagram (ERD) and concluding with successful frontend and backend development. We gained valuable insights and practical experience in database design, SQL, and user interface design.

Initially, we meticulously crafted the ERD, refining it by identifying weak entities, fixing constraint notations, and adjusting primary keys. We then translated the ERD into relational schemas, defining keys for each schema and creating tables in SQL.

Essential features such as user login, registration, ticket booking, and profile management were designed and implemented.

Integrating frontend and backend components presented challenges, particularly in deciding the best methods for database connectivity. Despite difficulties, we successfully connected many pieces but encountered limitations due to time constraints.

Takeaways

1. Applying classroom concepts to real-life scenarios shaped critical decisions in database design, significantly impacting the final product.
2. The decision to model payment as a weak entity dependent on the Order entity altered the database significantly.
3. Careful consideration of entity attributes led to the identification of candidate and foreign keys, enhancing relational schema accuracy.
4. Backend development highlighted the importance of optimizing database performance through indexing and query optimization.
5. Integration phase emphasized seamless communication between frontend and backend components through integration tests.
6. Try wild explorations to the point of failure.
